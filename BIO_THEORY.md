# Теория и смысл задач по биоинформатике

Этот файл дополняет `README.md`: здесь меньше про реализацию в Java и больше про то, что происходит в задачах с биологической и алгоритмической точки зрения.

Проект решает набор классических задач Rosalind. Почти все они сводят биологический объект к строке, графу, матрице или дереву:

- ДНК представляется строкой над алфавитом `{A, C, G, T}`.
- Белок представляется строкой аминокислот.
- Фрагменты секвенирования представляются короткими строками, то есть `k`-мерами.
- Сходство последовательностей измеряется расстояниями, score-матрицами и выравниваниями.
- Сборка генома переводится в задачу о графах де Брёйна и эйлеровых путях.
- Поиск мотивов переводится в перебор похожих `k`-меров и вероятностные профили.
- Филогенетика переводится в построение дерева по матрице расстояний.

## Базовая биологическая картина

### ДНК как строка

ДНК состоит из нуклеотидов:

- `A` — аденин;
- `C` — цитозин;
- `G` — гуанин;
- `T` — тимин.

В задачах ДНК почти всегда рассматривается как обычная строка. Например, геном `ACGTACG` можно резать на подстроки, сравнивать с другой строкой, искать в нем повторы или строить по нему граф.

Такое упрощение очень полезно: реальные биологические процессы сложнее, но многие вычислительные задачи начинаются именно со строкового представления.

### Белки и аминокислотные строки

Белки состоят из аминокислот. В задачах на выравнивание белков строки могут содержать символы вроде `A`, `R`, `N`, `D`, `C`, `Q`, `E`, `G` и так далее.

Для белков простое правило "совпало / не совпало" часто слишком грубое. Одни замены аминокислот биологически более допустимы, другие менее допустимы. Поэтому используются scoring matrices:

- `BLOSUM62` — часто используется для глобального выравнивания белков;
- `PAM250` — часто используется для локального выравнивания более далеких белковых последовательностей.

Матрица говорит, сколько очков дать за пару аминокислот. Похожая замена может иметь положительный или небольшой отрицательный score, а редкая и биологически маловероятная — большой штраф.

### k-меры

`k`-мер — это подстрока длины `k`.

Например, для строки `TATGGGGTGC` и `k = 3` все 3-меры:

```text
TAT
ATG
TGG
GGG
GGG
GGT
GTG
TGC
```

В биоинформатике `k`-меры важны потому, что секвенатор обычно не читает весь геном целиком. Он выдает много коротких фрагментов, а потом из этих фрагментов нужно восстановить исходную последовательность или найти в ней закономерности.

### Мотивы

Мотив — это короткий фрагмент ДНК, который повторяется в разных последовательностях и может иметь биологический смысл. Например, мотив может быть участком, куда садится транскрипционный фактор и регулирует работу гена.

В реальных данных мотивы редко совпадают идеально:

- в разных организмах могли накопиться мутации;
- секвенирование может содержать ошибки;
- один и тот же функциональный сигнал может допускать небольшие вариации.

Поэтому задачи про мотивы часто используют Hamming distance и разрешают несколько mismatches.

### Выравнивание

Выравнивание показывает, какие символы двух последовательностей соответствуют друг другу. В выравнивание можно вставлять gap-и:

```text
PLEASANTLY
-MEA--N-LY
```

Gap обычно интерпретируется как индел: insertion или deletion, то есть вставка или удаление фрагмента в ходе эволюции.

Есть несколько разных смыслов выравнивания:

- global alignment — сравнить две строки целиком;
- local alignment — найти самые похожие подстроки;
- fitting alignment — "вписать" короткую строку в участок длинной;
- affine gap alignment — штрафовать открытие gap-а сильнее, чем его продолжение.

### Филогенетическое дерево

Филогенетическое дерево показывает предполагаемые эволюционные связи между объектами: видами, генами, белками или другими таксонами.

В задачах UPGMA и Neighbor Joining входом является матрица расстояний. Чем меньше расстояние между двумя объектами, тем более похожими они считаются. Алгоритм пытается построить дерево, длины ребер которого объясняют эти расстояния.

## Algorithms / Strings

### Compute the Hamming Distance Between Two Strings

**Биологический смысл.**  
Hamming distance считает число позиций, в которых две строки одинаковой длины различаются. Для ДНК это простая модель количества точечных замен.

Пример:

```text
CGAAT
CGGAC
  ^ ^
```

Расстояние равно `2`.

**Что происходит в задаче.**

Даны две ДНК-строки одинаковой длины. Нужно пройти по ним слева направо и посчитать позиции, где символы не совпадают.

**Алгоритм.**

1. Заводим счетчик `distance = 0`.
2. Для каждой позиции `i` сравниваем `s[i]` и `t[i]`.
3. Если символы разные, увеличиваем счетчик.
4. Возвращаем счетчик.

**Сложность.**

- Время: `O(n)`.
- Память: `O(1)`.

### Compute the Edit Distance Between Two Strings

**Биологический смысл.**  
Edit distance, или расстояние Левенштейна, измеряет минимальное число операций, нужных для превращения одной строки в другую:

- вставка символа;
- удаление символа;
- замена символа.

Для биологических последовательностей это грубая модель эволюционных изменений: substitutions, insertions и deletions.

**Что происходит в задаче.**

Даны две аминокислотные строки. Нужно найти минимальное количество edit operations.

**Алгоритмическая идея.**

Используется динамическое программирование:

```text
dp[i][j] = edit distance между первыми i символами первой строки
           и первыми j символами второй строки
```

Переход:

- удалить символ из первой строки: `dp[i - 1][j] + 1`;
- вставить символ: `dp[i][j - 1] + 1`;
- заменить или оставить символ: `dp[i - 1][j - 1] + cost`, где `cost = 0`, если символы равны, иначе `1`.

**Сложность.**

- Время: `O(nm)`.
- Память: обычно `O(nm)`, но можно хранить только две строки DP и получить `O(min(n, m))`.

## NGS и k-меры

NGS означает Next Generation Sequencing. Вместо чтения одного длинного генома секвенатор дает множество коротких reads. Поэтому многие задачи работают не с целым геномом напрямую, а с набором коротких фрагментов.

### Generate the k-mer Composition of a String

**Биологический смысл.**  
Если секвенатор прочитал много коротких фрагментов, то мы часто знаем набор маленьких кусочков, но не знаем их исходный порядок. `k`-mer composition — это модель такого набора кусочков.

**Что происходит в задаче.**

Даны `k` и строка `Text`. Нужно вывести все подстроки длины `k`.

Пример:

```text
Text = TATGGGGTGC
k = 3
```

Получаем:

```text
ATG
GGG
GGG
GGT
GTG
TAT
TGC
TGG
```

Rosalind обычно выводит их в лексикографическом порядке, потому что при секвенировании реальный порядок reads неизвестен.

**Алгоритм.**

1. Пройти по всем позициям `0..n-k`.
2. Взять `Text[i..i+k)`.
3. Добавить в список.
4. Отсортировать список, если это требуется форматом.

### Find the Most Frequent Words in a String

**Биологический смысл.**  
Частые `k`-меры могут указывать на важные повторяющиеся элементы генома. В учебных задачах это часто связано с поиском потенциальных регуляторных участков или областей, важных для репликации.

**Что происходит в задаче.**

Даны `Text` и `k`. Нужно найти все `k`-меры, которые встречаются максимальное число раз.

**Алгоритм.**

1. Перебрать все `k`-меры строки.
2. Посчитать частоты в `HashMap`.
3. Найти максимальную частоту.
4. Вывести все `k`-меры с этой частотой.

**Важная деталь.**  
Если максимум достигается несколькими `k`-мерами, нужно вернуть все.

### Find Patterns Forming Clumps in a String

**Биологический смысл.**  
Clump — это локальное скопление повторов. Если короткий паттерн много раз встречается внутри небольшого окна генома, это может быть сигналом биологической функции.

Например, `TGCA` образует `(25, 3)`-clump, если в некотором участке длины `25` он встречается хотя бы `3` раза.

**Что происходит в задаче.**

Даны:

- `Genome`;
- длина паттерна `k`;
- длина окна `L`;
- порог повторов `t`.

Нужно найти все `k`-меры, которые в каком-то окне длины `L` встречаются минимум `t` раз.

**Алгоритмическая идея.**

Наивно можно для каждого окна заново считать все `k`-меры. Лучше использовать sliding window:

1. Посчитать частоты `k`-меров в первом окне.
2. Отметить те, у кого частота `>= t`.
3. Сдвинуть окно на один символ.
4. Уменьшить частоту выходящего `k`-мера.
5. Увеличить частоту входящего `k`-мера.
6. Повторять до конца генома.

## Motif Search

### Implement MotifEnumeration

**Биологический смысл.**  
Задача ищет общий мотив в нескольких ДНК-строках. Мотив не обязан встречаться идеально: допускается до `d` несовпадений.

Это похоже на ситуацию, когда разные гены регулируются одним и тем же фактором, но binding site немного отличается в каждой последовательности.

**Что происходит в задаче.**

Даны:

- `k` — длина мотива;
- `d` — максимально допустимое число mismatches;
- набор строк `Dna`.

Нужно найти все `k`-меры, которые встречаются в каждой строке из `Dna` с расстоянием Хэмминга не больше `d`.

**Алгоритм.**

1. Взять каждый `k`-мер из входных строк.
2. Сгенерировать всех соседей на расстоянии не больше `d`.
3. Для каждого кандидата проверить, есть ли он в каждой ДНК-строке с `<= d` mismatches.
4. Удалить дубликаты.

**Почему это дорого.**  
Количество возможных соседей быстро растет с `k` и `d`. Поэтому задача экспоненциальная по этим параметрам, но для учебных ограничений это нормально.

### Find a Median String

**Биологический смысл.**  
Median string — это паттерн, который в среднем лучше всего объясняет набор ДНК-строк. Он может вообще не встречаться в строках идеально, но быть наиболее близким к каждой из них.

**Формально.**

`d(Pattern, Text)` — минимальное Hamming distance между `Pattern` и любым `k`-мером в `Text`.

`d(Pattern, Dna)` — сумма таких расстояний по всем строкам.

Нужно найти `Pattern`, минимизирующий `d(Pattern, Dna)`.

**Алгоритм.**

1. Перебрать все `4^k` возможные строки длины `k` над `{A, C, G, T}`.
2. Для каждой строки посчитать сумму минимальных расстояний до всех `Dna`.
3. Выбрать строку с минимальной суммой.

**Смысл результата.**  
Это самый "центральный" мотив относительно набора последовательностей.

### Find a Profile-most Probable k-mer in a String

**Биологический смысл.**  
Profile matrix описывает вероятностный мотив. В каждой позиции она говорит, насколько вероятны `A`, `C`, `G`, `T`.

Например, если в позиции 1 вероятность `A` высокая, а вероятность `T` почти нулевая, то хороший кандидат на мотив скорее начнется с `A`.

**Что происходит в задаче.**

Даны:

- строка `Text`;
- длина `k`;
- матрица `Profile` размера `4 x k`.

Нужно найти `k`-мер из `Text` с максимальной вероятностью по профилю.

**Как считается вероятность.**

Для `k`-мера перемножаются вероятности его символов по позициям:

```text
P(ACG) = P(A in column 1) * P(C in column 2) * P(G in column 3)
```

Если несколько `k`-меров имеют одинаковую максимальную вероятность, выбирается первый встретившийся в `Text`.

## Выравнивание последовательностей

Выравнивание — центральный инструмент сравнения биологических последовательностей. Оно помогает понять:

- насколько похожи две последовательности;
- какие участки могли произойти от общего предка;
- где могли быть мутации, вставки или удаления;
- какие белковые участки могут иметь похожую функцию.

### Find a Highest-Scoring Fitting Alignment

**Биологический смысл.**  
Fitting alignment нужен, когда короткую последовательность нужно "вписать" в длинную.

Пример из условия: есть огромный белок и короткий домен. Нужно понять, где в длинном белке может находиться область, похожая на этот домен.

Global alignment здесь плох, потому что заставляет сравнивать всю длинную строку. Local alignment тоже не совсем то, потому что может взять только кусок от короткой строки. Fitting alignment требует использовать всю короткую строку, но разрешает выбрать лучший участок длинной.

**Scoring.**

- match: `+1`;
- mismatch: `-1`;
- indel/gap: `-1`.

**Алгоритмическая идея.**

Используется DP, похожее на global alignment:

- короткая строка должна быть выровнена полностью;
- длинная строка может иметь необязательный префикс и суффикс вне выравнивания.

Поэтому первая колонка часто инициализируется нулями: это означает, что можно бесплатно пропустить начало длинной строки.

### Find a Highest-Scoring Global Alignment of Two Strings

**Биологический смысл.**  
Global alignment сравнивает две последовательности целиком. Он подходит, когда строки примерно одной длины и предполагается, что они целиком гомологичны.

**В этой задаче.**

- Вход: две аминокислотные строки.
- Scoring matrix: `BLOSUM62`.
- Gap penalty: `5`.
- Выход: максимальный score и одно оптимальное выравнивание.

**Алгоритм Needleman-Wunsch.**

`dp[i][j]` — лучший score для префиксов `v[0..i)` и `w[0..j)`.

Переход:

- diagonal: выровнять `v[i-1]` с `w[j-1]`;
- down: поставить gap во второй строке;
- right: поставить gap в первой строке.

Для восстановления самого выравнивания хранится направление перехода или выполняется backtracking по DP-таблице.

### Find a Highest-Scoring Local Alignment of Two Strings

**Биологический смысл.**  
Local alignment ищет самые похожие участки двух последовательностей. Это полезно, когда две длинные последовательности похожи только в одном домене или функциональном участке.

**В этой задаче.**

- Вход: две аминокислотные строки.
- Scoring matrix: `PAM250`.
- Indel penalty: `5`.

**Алгоритм Smith-Waterman.**

Похож на global alignment, но в каждой клетке разрешен переход в `0`:

```text
dp[i][j] = max(0, diagonal, down, right)
```

Зачем нужен `0`: если накопленный score стал плохим, локальное выравнивание может начаться заново.

Backtracking начинается из глобального максимума в таблице и заканчивается, когда пришли в клетку со score `0`.

### Align Two Strings Using Affine Gap Penalties

**Биологический смысл.**  
Один длинный gap обычно биологически правдоподобнее, чем много независимых коротких gap-ов подряд. Например, удаление участка из 5 аминокислот могло произойти одним событием.

Поэтому используется affine gap penalty:

```text
penalty(k) = sigma + epsilon * (k - 1)
```

где:

- `sigma` — штраф за открытие gap-а;
- `epsilon` — штраф за продолжение gap-а.

В задаче:

- gap opening: `11`;
- gap extension: `1`;
- scoring matrix: `BLOSUM62`.

**Алгоритмическая идея.**

Вместо одной DP-таблицы используются три:

- `middle` — последнее действие было match/mismatch;
- `lower` — сейчас продолжается gap в одной строке;
- `upper` — сейчас продолжается gap в другой строке.

Так алгоритм различает два случая:

- открыть новый gap и заплатить большой штраф;
- продолжить уже открытый gap и заплатить маленький штраф.

## Индексация и поиск паттернов

Когда нужно искать много коротких строк в большом геноме, наивный перебор становится медленным. Для ускорения строятся структуры данных: trie, suffix array, Burrows-Wheeler transform.

### Construct a Trie from a Collection of Patterns

**Биологический смысл.**  
Есть набор reads или паттернов, которые нужно искать в геноме. Вместо отдельного поиска каждого паттерна можно объединить их общие префиксы в одно дерево.

**Trie.**

Trie — это префиксное дерево:

- корень соответствует пустой строке;
- ребра подписаны символами;
- путь от корня до узла задает префикс;
- путь до листа задает один из паттернов.

**Что происходит в задаче.**

Дан набор строк `Patterns`. Нужно построить trie и вывести его ребра.

**Алгоритм.**

1. Начать с одного корня.
2. Для каждого паттерна идти по символам.
3. Если нужное ребро уже есть, перейти по нему.
4. Если ребра нет, создать новый узел и новое ребро.

### Implement TrieMatching

**Биологический смысл.**  
Теперь trie используется для поиска всех вхождений паттернов в большом тексте, например в геноме.

**Что происходит в задаче.**

Даны:

- `Text`;
- набор `Patterns`.

Нужно вернуть все стартовые позиции в `Text`, где начинается хотя бы один паттерн.

**Алгоритм.**

1. Построить trie по `Patterns`.
2. Для каждой позиции `i` в `Text` запустить prefix matching.
3. Двигаться вниз по trie, сравнивая символы текста с подписями ребер.
4. Если дошли до terminal/leaf-узла, значит паттерн найден с позиции `i`.
5. Если подходящего ребра нет, совпадения с этой позиции нет.

### Construct the Burrows-Wheeler Transform of a String

**Биологический смысл.**  
BWT сам по себе не является биологической моделью, но это важный инструмент для компактного хранения и быстрого поиска в больших геномах. На идеях BWT основаны FM-index и многие read aligners.

**Что происходит в задаче.**

Дана строка `Text`, обычно с терминальным символом `$`. Нужно построить `BWT(Text)`.

**Алгоритм.**

1. Построить все циклические сдвиги строки.
2. Отсортировать их лексикографически.
3. Взять последний символ каждой строки в отсортированной матрице.
4. Склеить эти символы — это и есть BWT.

**Интуиция.**  
BWT часто группирует одинаковые символы рядом, поэтому после него строка хорошо сжимается и позволяет эффективно искать паттерны.

### Reconstruct a String from its Burrows-Wheeler Transform

**Что происходит в задаче.**

Дан BWT-результат. Нужно восстановить исходную строку.

**Ключевая идея.**

В BWT есть last column — данная строка transform. First column получается сортировкой символов last column.

Если одинаковые символы пронумеровать по порядку появления, например:

```text
A1, A2, A3
```

то можно построить last-to-first mapping. Он показывает, какая строка матрицы соответствует какому символу при переходе из последнего столбца в первый.

Идя по этому отображению от `$`, можно восстановить исходный текст.

### Pattern Matching with the Suffix Array

**Биологический смысл.**  
Suffix array позволяет быстро искать паттерны в большом тексте. Это важно для поиска reads в геноме.

**Suffix array.**

Suffix array — это массив стартовых позиций всех суффиксов строки в лексикографическом порядке.

Например, если все суффиксы отсортированы, то все суффиксы, начинающиеся с паттерна `P`, образуют непрерывный диапазон. Этот диапазон можно найти бинарным поиском.

**Что происходит в задаче.**

Даны `Text` и набор `Patterns`. Нужно вывести все позиции, где встречается хотя бы один паттерн.

**Алгоритм.**

1. Построить suffix array для `Text`.
2. Для каждого паттерна бинарным поиском найти диапазон суффиксов, начинающихся с этого паттерна.
3. Собрать стартовые позиции из найденного диапазона.
4. Удалить дубликаты и вывести позиции.

## Сборка генома и графы де Брёйна

При секвенировании мы часто получаем набор reads, но не знаем, как они были расположены в исходном геноме. Один из подходов к сборке — построить граф де Брёйна.

Главная идея:

- `k`-мер становится ребром;
- его prefix длины `k-1` становится начальной вершиной;
- его suffix длины `k-1` становится конечной вершиной;
- восстановление генома превращается в прохождение по каждому ребру ровно один раз.

### Construct the De Bruijn Graph of a String

**Что происходит в задаче.**

Даны `k` и строка `Text`. Нужно построить `DeBruijn_k(Text)`.

Для каждого `k`-мера:

```text
kmer = Text[i..i+k)
prefix = kmer[0..k-1)
suffix = kmer[1..k)
```

Добавляется ребро:

```text
prefix -> suffix
```

Одинаковые вершины "склеиваются": если один и тот же `(k-1)`-мер встречается много раз, это одна вершина графа.

### Construct the De Bruijn Graph of a Collection of k-mers

**Что отличается от предыдущей задачи.**

Теперь исходная строка неизвестна. Есть только набор `k`-меров `Patterns`.

Для каждого `k`-мера строится ребро `prefix -> suffix`. Если `k`-мер повторяется, кратность ребра сохраняется, потому что повтор может быть биологически важен.

**Биологический смысл.**  
Это ближе к реальной сборке генома: секвенатор дал reads, а исходный порядок нужно восстановить.

### Find an Eulerian Cycle in a Graph

**Графовая идея.**  
Эйлеров цикл проходит по каждому ребру графа ровно один раз и возвращается в стартовую вершину.

**Почему это важно для сборки генома.**

В графе де Брёйна каждое ребро соответствует `k`-меру/read. Если пройти по всем ребрам ровно один раз, мы используем все фрагменты для восстановления последовательности.

**Алгоритм Hierholzer.**

1. Начать с любой вершины.
2. Идти по неиспользованным ребрам, пока можно.
3. Получится цикл.
4. Если в цикле есть вершина с неиспользованными ребрами, начать из нее новый цикл.
5. Вставить новый цикл в старый.
6. Повторять, пока все ребра не использованы.

### Find an Eulerian Path in a Graph

**Отличие от цикла.**  
Эйлеров путь тоже проходит по каждому ребру ровно один раз, но не обязан возвращаться в начало.

В ориентированном графе для эйлерова пути обычно:

- стартовая вершина имеет `outDegree = inDegree + 1`;
- конечная вершина имеет `inDegree = outDegree + 1`;
- остальные вершины имеют равные входящие и исходящие степени.

**Алгоритм.**

Можно использовать тот же Hierholzer, но стартовать из вершины с лишним исходящим ребром.

### Reconstruct a String from its k-mer Composition

**Что происходит в задаче.**

Даны `k` и набор `k`-меров. Нужно восстановить строку, чья `k`-mer composition совпадает с этим набором.

**Связь задач.**

Эта задача собирается из трех предыдущих:

1. Построить de Bruijn graph по `k`-мерам.
2. Найти Eulerian path.
3. Превратить путь обратно в строку.

**Как путь превращается в строку.**

Если путь проходит по вершинам:

```text
ATG -> TGG -> GGC
```

то первая вершина берется целиком:

```text
ATG
```

А дальше от каждой следующей вершины добавляется последний символ:

```text
ATG + G + C = ATGGC
```

## Филогенетика

Филогенетические задачи строят дерево по матрице расстояний. Листья дерева — исходные объекты. Внутренние узлы — гипотетические предки или точки расхождения.

### Implement UPGMA

**Биологический смысл.**  
UPGMA строит ультраметрическое дерево. Это означает, что все листья находятся на одинаковом расстоянии от корня. Биологически это соответствует предположению о molecular clock: скорость эволюции примерно одинаковая во всех ветвях.

Это сильное предположение. Если скорости эволюции различаются, UPGMA может дать неверную топологию.

**Что происходит в задаче.**

Дана матрица расстояний между `n` объектами. Нужно построить дерево.

**Алгоритм.**

1. Каждый объект начинает как отдельный кластер.
2. На каждом шаге выбираются два ближайших кластера.
3. Они объединяются в новый кластер.
4. Возраст нового узла равен половине расстояния между объединяемыми кластерами.
5. Расстояние от нового кластера до остальных считается как среднее с учетом размеров кластеров:

```text
D(Cnew, C) = (|Ci| * D(Ci, C) + |Cj| * D(Cj, C)) / (|Ci| + |Cj|)
```

6. Процесс продолжается, пока не останется один кластер.

**Длины ребер.**

После построения дерева длина ребра от родителя к ребенку:

```text
Age(parent) - Age(child)
```

Листья имеют возраст `0`.

### Implement the Neighbor Joining Algorithm

**Биологический смысл.**  
Neighbor Joining тоже строит филогенетическое дерево по матрице расстояний, но не требует ультраметричности. Это делает его более гибким, чем UPGMA, когда разные линии эволюционировали с разной скоростью.

**Главная идея.**

Алгоритм ищет пару "соседей", которые выгодно соединить через новый внутренний узел. При этом он учитывает не только прямое расстояние между двумя объектами, но и их суммарную удаленность от всех остальных.

**Neighbor-joining matrix.**

Для матрицы расстояний `D` строится матрица:

```text
D'[i][j] = (n - 2) * D[i][j] - TotalDistance(i) - TotalDistance(j)
```

где:

```text
TotalDistance(i) = сумма D[i][k] по всем k
```

Минимальный недиагональный элемент `D'` указывает пару `i, j`, которую нужно объединить.

**Длины ветвей.**

После выбора пары считаются:

```text
Delta = (TotalDistance(i) - TotalDistance(j)) / (n - 2)
limbLength_i = 0.5 * (D[i][j] + Delta)
limbLength_j = 0.5 * (D[i][j] - Delta)
```

Эти значения становятся длинами ребер от нового внутреннего узла к `i` и `j`.

**Сжатие матрицы.**

Создается новый узел `m`. Его расстояние до любого другого узла `k`:

```text
D[k][m] = D[m][k] = 0.5 * (D[k][i] + D[k][j] - D[i][j])
```

После этого строки и столбцы `i` и `j` удаляются, а `m` добавляется. Алгоритм рекурсивно продолжается для `n - 1` узлов.

**Базовый случай.**

Если осталось два узла, между ними добавляется ребро длины `D[1][2]`.

**Почему внутренние узлы нумеруются с `n`.**

В условиях Rosalind листья должны иметь метки `0..n-1`, потому что они соответствуют исходным строкам/таксонам. Новые внутренние узлы не были во входе, поэтому им дают новые последовательные номера начиная с `n`.
